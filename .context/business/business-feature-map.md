## 10. Discovery gaps

Greenfield project gaps surfaced while mapping PRD/SRS to features. None block MVP; flagged for `/project-bootstrap` and `/product-management` to resolve.

### Gap #1 ŌĆö Flag naming inconsistency for bidirectional Jira sync

`phase2.jira_bidirectional` is named as a Phase-2 flag in the architecture entity table, but `executive-summary.md ┬¦5` and `mvp-scope.md ┬¦3` list "Bidirectional Jira sync" as a Phase-3 feature. Resolution path: rename to `phase3.jira_bidirectional` OR move the feature into Phase 2 scope. **Ask**: founder ŌĆö which phase does bidirectional Jira sync target? Action item for `/product-management` when creating the Phase-2 epic.

### Gap #2 ŌĆö Workspace invite lifecycle endpoints missing

{{PROJECT_KEY}}-003 covers invite creation but `.context/SRS/api-contracts.yaml` exposes no GET / resend / revoke endpoints for `workspace_invites`. The 24h-expiry behavior implies users will hit expired-invite cases (covered in user-journeys.md ┬¦1 edge case "Workspace invite expired"). **Ask**: should `/workspaces/{id}/invites` support GET-list + `POST /workspaces/{id}/invites/{invite_id}/resend` + `DELETE` in MVP, or defer to Phase 2? Likely MVP-needed for invite UX.

### Gap #3 ŌĆö Settings / integrations / environments have UI but no public API

`environments` and `integrations` tables exist in the entity table; both surface in Settings UI screens (5.1), but neither has a public endpoint in `api-contracts.yaml v1.0`. {{PROJECT_KEY}}-019 references `environment` enum, {{PROJECT_KEY}}-009/{{PROJECT_KEY}}-028 reference Jira credentials retrieved from "Workspace integration config" ŌĆö but the management API for these is absent. **Ask**: are these Settings-only (server actions, never exposed as Bearer-protected REST) or should they be added to `/api/v1/...` for CLI / agentic use? Decision needed before `/project-bootstrap` scaffolds API routes.

### Gap #4 ŌĆö Read/Update/Delete endpoints for User Stories + Acceptance Criteria

`api-contracts.yaml v1.0` documents only `POST /user-stories` and `POST /acceptance-criteria` ŌĆö no GET / PATCH / DELETE. {{PROJECT_KEY}}-035 generic ("All FRs 001ŌĆō032 exposed under `/api/v1/...`") implies they exist but the contract file is incomplete. **Ask**: surface these endpoints in `api-contracts.yaml` before `/project-bootstrap` so OpenAPI consumers (Karim persona) don't hit 404s. Action item for `/project-foundation` Phase 4 wrap-up or `/project-bootstrap` Step 1.

### Gap #5 ŌĆö Sweeper cron for abandoned Runs

`user-journeys.md ┬¦3 "Agent abandons (process killed)"` mentions "a sweeper cron times it out after a configurable max_duration (default 4h)" ŌĆö but no FR documents this behavior. **Ask**: codify as cross-cutting FR (e.g. {{PROJECT_KEY}}-041 Run abandonment sweeper) and decide whether `max_duration` is per-Project, per-Workspace, or global. Action item for `/product-management` before MVP launch.

### Gap #6 ŌĆö Workspace plan field has no toggle / billing endpoint

`workspaces.plan` Ōłł {community, cloud, enterprise} is the license-gate mechanism (architecture-specs ┬¦1) but no FR or endpoint documents how it's set. Implicit assumption: provisioned out-of-band (manual SQL by founder for design-partners in MVP; Stripe webhook later). **Ask**: confirm MVP plan = out-of-band provisioning; defer Stripe / billing to Phase 2.

### Gap #7 ŌĆö Search across `acceptance_criteria` not in API

{{PROJECT_KEY}}-031 (command palette) enumerates `entity_type` enum that includes `acceptance_criterion`, but no dedicated `/acceptance-criteria` search endpoint exists. Likely served by the unified `/search` endpoint. **Confirm**: `/search` covers AC entity; no separate endpoint needed.

### Gap #8 ŌĆö Markdown sanitization library not pinned

`architecture-specs ┬¦5` mentions "Markdown content sanitized via `rehype-sanitize` with an allowlist". Listed as a planned dependency (not in section 6 integrations table). **Ask**: add `rehype-sanitize` + Markdown rendering pipeline to project-bootstrap's frontend setup ŌĆö and confirm Mermaid renderer allowlist scope.

### Gap #9 ŌĆö Open-source license not yet selected

`business-model.md ┬¦pendientes-a-iterar` lists "Apache 2.0 vs MIT" as undecided before launch. Affects the Community edition shipping artifact. Not blocking MVP feature work but blocking marketing / repo public-ization.
