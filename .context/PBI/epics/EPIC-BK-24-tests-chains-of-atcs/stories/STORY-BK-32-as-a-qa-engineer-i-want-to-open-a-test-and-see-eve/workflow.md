# BK-32 — Workflow

> Jira field: `customfield_10161` · [View in Jira](https://upexgalaxy67.atlassian.net/browse/BK-32)

1. From the list of Tests, I pick a Test such as "Checkout - Guest Purchase" and open it.
2. The Test opens in an expanded view with a header showing its name and a count like "3 ATCs".
3. I scroll down and see each chained ATC laid out in running order, numbered by its position.
4. Inside each ATC, I read its ordered steps first, then the assertions that will be checked.
5. I confirm the sequence and content match what I intend to validate — everything is read-only, so I cannot accidentally change anything.
6. If the Test has no ATCs yet, I instead see an empty-state message prompting me to add ATCs, and I leave to go assemble the chain.
7. Once I am satisfied with what the Test covers, I leave the view ready to start a run.
